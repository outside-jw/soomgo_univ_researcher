# 실험 참가자용 질문 시나리오

## 📋 개요

이 문서는 CPS 스캐폴딩 에이전트를 활용한 실험에서 참가자들이 경험하게 될 질문 시나리오를 정리한 것입니다.

**시스템 설정:**
- 도전_이해 단계: 최대 6턴
- 아이디어_생성 단계: 최대 8턴
- 실행_준비 단계: 최대 6턴
- 총 최대 대화: 20턴

---

## 🎯 1단계: 도전 이해 (최대 6턴)

### 참가자 초기 입력
```
과제 제시문을 그대로 입력하거나 요약하여 입력
예: "초임교사로 방송 업무와 학급 담임을 하면서 수업 준비 시간이 부족하고,
    학생들의 수업 참여도가 낮아지는 문제를 해결하고 싶습니다."
```

### 턴 1: 점검 - 과제 익숙함
**에이전트 질문:**
> 해당 문제가 얼마나 익숙하게 느껴지나요? 그 이유는 무엇인가요?

**예상 참가자 응답 예시:**
- "교생 실습 때 비슷한 경험이 있어서 어느 정도 익숙합니다."
- "처음 겪는 상황이라 낯설게 느껴집니다."
- "이론으로는 배웠지만 실제로 경험하지 못해서 반만 익숙합니다."

### 턴 2: 점검 - 과제 난이도
**에이전트 질문:**
> 해당 문제의 난이도는 어느 정도라고 판단되나요? 그 이유는 무엇인가요?

**예상 참가자 응답 예시:**
- "여러 변수가 얽혀있어서 난이도가 높다고 생각합니다."
- "단계적으로 접근하면 해결 가능할 것 같아 중간 정도입니다."
- "시간과 자원이 제한되어 있어서 어렵습니다."

### 턴 3: 점검 - 자기효능감
**에이전트 질문:**
> 해당 문제를 얼마나 잘 해결할 수 있을 것 같나요?

**예상 참가자 응답 예시:**
- "노력하면 70% 정도는 해결할 수 있을 것 같습니다."
- "완전히 해결하기는 어렵지만 개선은 가능할 것 같습니다."
- "아직 확신이 없지만 최선을 다하겠습니다."

### 턴 4: 조절 - 자료 탐색
**에이전트 질문:**
> 문제 해결을 위해 제시문 및 예시를 더 자세히 검토해 볼까요?

**예상 참가자 응답 예시:**
- "네, 학생들이 지루해하는 구체적인 시점을 다시 살펴보겠습니다."
- "방송 업무와 수업 준비의 시간 충돌 부분을 더 분석하겠습니다."
- "학생들과의 관계 변화 시점을 다시 확인하겠습니다."

### 턴 5: 조절 - 이해 확인
**에이전트 질문:**
> 해당 문제와 예시를 충분히 이해했다고 생각하나요?

**예상 참가자 응답 예시:**
- "네, 핵심 원인 3가지를 파악했습니다."
- "아직 부족한 부분이 있는 것 같습니다."
- "대부분 이해했지만 한 가지 더 확인하고 싶습니다."

### 턴 6: 지식 - 경험 활용
**에이전트 질문:**
> 이전에 비슷한 문제를 해결해 본 경험이 있나요? 어떤 해결책이 효과적이었나요?

**예상 참가자 응답 예시:**
- "교생 실습 때 학생 참여를 높이기 위해 게임형 활동을 도입했던 것이 효과적이었습니다."
- "특별한 경험은 없지만 수업 관찰 시 상호작용이 많은 수업이 효과적이었습니다."

**⚠️ 6턴 도달 → 자동 전환**
> "도전_이해 단계의 최대 대화 턴 수(6턴)에 도달했습니다. 이제 아이디어_생성 단계로 진행합니다."

---

## 💡 2단계: 아이디어 생성 (최대 8턴)

### 턴 1: 지식 - 전략 선택
**에이전트 질문:**
> 해당 문제를 해결하기 위한 아이디어 생성 전략(예: 브레인스토밍)으로 어떤 것들이 있을까요? 가장 효과적이라고 판단되는 전략을 선택해 아이디어를 생성해볼까요?

**예상 참가자 응답 예시:**
- "브레인스토밍으로 다양한 수업 방법을 떠올려보겠습니다."
- "SCAMPER 기법을 사용해서 기존 수업을 변형해보겠습니다."
- "마인드맵으로 문제 원인별 해결책을 도출하겠습니다."

### 턴 2-4: 점검 - 아이디어 평가
**에이전트 질문:**
> 이 아이디어는 문제 해결 목표 달성에 얼마나 부합하나요?

**예상 참가자 응답 예시:**
- "플립러닝 도입: 수업 준비 시간 단축과 학생 참여 증대에 70% 부합합니다."
- "학생 중심 프로젝트 수업: 참여도 향상에는 좋지만 준비 시간이 더 필요합니다."
- "업무 분담 요청: 근본적 해결은 아니지만 시간 확보에 도움됩니다."

### 턴 5: 점검 - 다양성 확인
**에이전트 질문:**
> 현재 아이디어 수와 다양성은 충분하다고 느끼시나요?

**예상 참가자 응답 예시:**
- "5개 정도 나왔는데 더 창의적인 아이디어가 필요할 것 같습니다."
- "충분히 다양하게 나온 것 같습니다."

### 턴 6: 점검 - 강점 비교
**에이전트 질문:**
> 기존 아이디어와 비교했을 때, 이 아이디어의 강점은 무엇인가요?

**예상 참가자 응답 예시:**
- "디지털 도구 활용은 준비 시간이 짧고 학생 흥미도가 높습니다."
- "동료 교사 협력은 실행 가능성이 가장 높습니다."

### 턴 7: 조절 - 아이디어 발전
**에이전트 질문:**
> 지금 떠올린 아이디어를 더 발전시킬 수 있을까요?

**예상 참가자 응답 예시:**
- "짧은 동영상 활용에서 → 학생이 직접 만드는 동영상 프로젝트로 발전시킬 수 있습니다."
- "개인 활동에서 → 모둠 활동으로 확장할 수 있습니다."

### 턴 8: 조절 - 전략 변경
**에이전트 질문:**
> 현재 아이디어가 잘 떠오르지 않는다면, 아이디어 생성 전략을 바꿔볼까요?

**예상 참가자 응답 예시:**
- "역발상으로 생각해보겠습니다. '수업을 더 지루하게 만들려면?' → 반대로 하면 됩니다."
- "다른 교과 수업 사례를 참고하겠습니다."

**⚠️ 8턴 도달 → 자동 전환**
> "아이디어_생성 단계의 최대 대화 턴 수(8턴)에 도달했습니다. 이제 실행_준비 단계로 진행합니다."

---

## 🚀 3단계: 실행 준비 (최대 6턴)

### 턴 1: 조절 - 아이디어 선택
**에이전트 질문:**
> 도출한 아이디어 중 가장 창의적이면서 실행 가능한 것은 무엇인가요?

**예상 참가자 응답 예시:**
- "플립러닝과 짧은 퀴즈 게임을 결합한 방법을 선택하겠습니다."
- "학생 주도 프로젝트와 동료 교사 협력을 선택하겠습니다."

### 턴 2: 점검 - 창의성 평가
**에이전트 질문:**
> 도출된 아이디어들을 창의성과 실행 가능성 관점에서 평가해볼까요?

**예상 참가자 응답 예시:**
- "5개 아이디어 중 3개는 독창적이지만 실행이 어렵고, 2개는 현실적입니다."
- "창의성: A>B>C, 실행 가능성: C>B>A 순서입니다."

### 턴 3-4: 점검 - 전략 효과성
**에이전트 질문:**
> 문제 해결 과정을 돌아봤을 때, 아이디어 생성을 위해 효과적이었던 전략과 그렇지 못했던 전략은 무엇인가요?

**예상 참가자 응답 예시:**
- "브레인스토밍이 효과적이었고, 너무 이론적으로 접근한 것은 비효과적이었습니다."
- "학생 관점에서 생각해본 것이 효과적이었습니다."

### 턴 5: 지식 - 학습 성찰
**에이전트 질문:**
> 이번 문제 해결을 통해 새롭게 배운 점은 무엇인가요? 앞으로 비슷한 문제를 해결할 때 이 경험을 어떻게 활용할 수 있을까요?

**예상 참가자 응답 예시:**
- "문제의 근본 원인을 먼저 파악하는 것이 중요함을 배웠습니다."
- "학생 관점에서 생각하면 더 현실적인 해결책이 나온다는 것을 알았습니다."

### 턴 6: 지식 - 메타인지 갱신
**에이전트 질문:**
> 이번 문제 해결을 수행하며 자신의 사고나 전략에 대해 새롭게 알게 된 점이 있나요?

**예상 참가자 응답 예시:**
- "처음에는 조급하게 해결책을 찾으려 했는데, 단계적 접근이 더 효과적임을 알았습니다."
- "다양한 관점에서 생각하는 것이 제 강점임을 발견했습니다."

**⚠️ 6턴 도달 → 종료**
> "실행_준비 단계의 최대 대화 턴 수(6턴)에 도달했습니다. 문제 해결 과정을 마무리합니다."

```
도전_이해: 3턴 (자발적 전환)
아이디어_생성: 5턴 (자발적 전환)

총 12턴
```

아이디어_생성: 7턴 (충분한 아이디어 도출)

총 17턴

총 20턴 (최대)

## 🎯 실험 참가자 안내 사항

### 1. 시작 전 안내
- "이 시스템은 여러분의 창의적 문제해결을 돕는 AI 에이전트입니다."
- "각 단계마다 대화 턴 수가 제한되어 있으며, 제한에 도달하면 자동으로 다음 단계로 넘어갑니다."
- "화면 상단에서 현재 단계와 남은 턴 수를 확인할 수 있습니다."

### 2. 응답 작성 팁
- 구체적으로 답변할수록 더 적절한 질문을 받을 수 있습니다
- 한 번에 여러 아이디어를 제시하지 말고, 하나씩 제시하세요
- "잘 모르겠습니다"보다는 현재 생각을 솔직하게 표현하세요

### 3. 단계 전환 안내
- 자발적 전환: 충분히 탐색했다고 판단되면 다음 단계로 진행 의사를 밝히세요
- 강제 전환: 턴 제한에 도달하면 자동으로 다음 단계로 넘어갑니다
- 전환 시 알림 메시지가 표시됩니다

### 4. 예상 소요 시간
- 최소: 15-20분 (빠른 진행)
- 표준: 25-35분 (적정 탐색)
- 최대: 40-50분 (최대 턴 사용)

## 📝 연구자용 관찰 포인트

### 1. 턴 제한의 효과
- [ ] 참가자가 턴 제한을 인지하고 있는가?
- [ ] 강제 전환이 발생했는가? (몇 번째 단계에서?)
- [ ] 강제 전환 시 참가자의 반응은?

### 2. 질문의 적절성
- [ ] 각 단계별 질문이 사고를 촉진했는가?
- [ ] 메타인지 요소(점검/조절/지식)가 골고루 다뤄졌는가?
- [ ] 질문이 중복되거나 맥락에 맞지 않은 경우가 있었는가?

### 3. 참가자 경험
- [ ] 참가자가 CPS 3단계를 모두 경험했는가?
- [ ] 실행 준비 단계까지 도달했는가?
- [ ] 만족스러운 해결책을 도출했는가?

### 4. 시스템 성능
- [ ] 응답 시간이 적절했는가? (목표: 3초 이내)
- [ ] 단계 인식이 정확했는가? (목표: 85% 이상)
- [ ] UI가 직관적이고 사용하기 쉬웠는가?

## 🔧 문제 발생 시 대응

### 시나리오 1: 에이전트가 같은 질문 반복
**원인:** 참가자 응답이 너무 짧거나 모호함
**대응:** 참가자에게 더 구체적인 답변 요청

### 시나리오 2: 턴 제한 전에 진행이 너무 느림
**원인:** 에이전트가 단계 전환을 과도하게 지연
**대응:** 6턴 제한이 자동으로 해결함 (강제 전환)

### 시나리오 3: 참가자가 혼란스러워함
**원인:** CPS 단계에 대한 이해 부족
**대응:**
- 화면의 진행 상황 표시를 참고하도록 안내
- "지금은 문제를 이해하는 단계입니다" 등의 맥락 제공

---

이 시나리오는 실제 실험 진행 시 참고 자료로 활용하시면 됩니다. 실제 에이전트는 학습자의 응답에 따라 동적으로 질문을 생성하므로, 위의 순서나 질문이 정확히 일치하지 않을 수 있습니다.
